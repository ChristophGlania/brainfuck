Brainfuck Editor

The Brainfuck Editor enhances the standard text editor with a few functions.
  • Syntax Highlighting
    • The Brainfuck key characters + - < > . , [ ].
    • A sequence of numbers (exclamation mark for negative values) separated by a semicolon are recognized as input parameters for a template.
    • The special sequence [-][] will not be executed by Brainfuck within the second pair of bracket and thus being marked as comment. This construct is safe as long as all brackets are balanced but will delete the contents at the actual memory location.
    • All other characters are ignored and assumed to be inline comments.
  • Typing
    Typing an opening bracket [ can be completed automatically by a corresponding closing bracket ].
    Indentation is automatically propagated to the next line.
  • Content Assist
    Content assist can be triggered any time (Default is CTRL-SPACE) within code. This will show the applicable templates. Content assist can then be toggled to display the key characters.
    Some of the templates operate on non-Brainfuck characters preceding the trigger location as customized input to the template. Valid input parameters will be colored separately and the templates are sorted according to their presumed acceptance of number of parameters matching the number of parameters at the trigger location.

    Example:
    3;2 will evaluate to two parameters 3 and 2 respectively. Templates which take two parameters will then show up on top of the completion proposals, then templates taking no parameter followed by the templates taking less than two parameters followed by the rest.

  • Error Checking
    Code is automatically checked for matching brackets.
